# doc/Technical/

Technical specifications — stack, functions, token budgets, build constraints,
API references. Part of the `CLAUDE-FULL.md` §1 doc structure.

- **Filename:** `technical-<version>-<date>.md` — version `M_m_p`, date
  `dd_mm_yy-hhmm` (`CLAUDE-FULL.md` §2). Example: `technical-0_1_0-21_05_26-2130.md`.
- **Update cadence:** per minor or major version bump (§9).
- **Confirmation gate:** G1 — version bumps need operator confirmation (§8).
- Superseded versions move to `doc/Archive/` unchanged; never overwrite (Rule N-3).
