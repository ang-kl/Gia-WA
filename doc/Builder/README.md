# doc/Builder/

Builder-lens / framework / persona docs — *how* the build is conducted
(patterns, working methods). Part of the `CLAUDE-FULL.md` §1 doc structure.

- **Filename:** `builder-<version>-<date>.md` — version `M_m_p`, date
  `dd_mm_yy-hhmm` (`CLAUDE-FULL.md` §2). Example: `builder-0_1_0-21_05_26-2130.md`.
- **Update cadence:** per session, if new build patterns are observed (§9).
- **Confirmation gate:** G3 — new patterns need operator approval (§8).
- Superseded versions move to `doc/Archive/` unchanged; never overwrite (Rule N-3).
