## Headline

**The port plan holds up.** `gia` is intensely active — journal entry #299,
~306 commits, **4 releases on the audit day alone** (`v0.61.96`→`v0.61.99`) —
but the recent work is feature/patch churn (transport-map polish, station
pins) *within* the architecture `§3` describes. No architectural divergence.
One genuine factual correction was needed (`§3.3`).

## Findings by section

### §3.1 Telegram surface — ACCURATE

`index.js` (~640 KB), the 5 Mini Apps (`web/cuisine`, `hawker`, `menu`,
`oversight`, `transport`), `twa-auth.js`, and `node-telegram-bot-api ^0.64.0`
are all present and intact.

### §3.2 AI integration — ACCURATE (verified carefully)

- `llm-client.js` is the **single Claude integration point** — it is the only
  file importing `@anthropic-ai/sdk` (`^0.40.0`), and its own header calls it
  "the single integration point for Claude."
- `nl-intent.js` → `llm.HAIKU_MODEL`; `prompt-builder.js` → `llm.SONNET_MODEL`
  (`claude-sonnet-4-6`, env-overridable via `ANTHROPIC_STAGE_A_MODEL`);
  `consultant.js` → `llm.HAIKU_MODEL`. Matches `§3.2`.
- `gemini-client.js` is the R.E.D path — now a multi-model fallback chain
  (`gemini-1.5-pro` → `2.0-flash` → `2.5-flash`, with a `claude-sonnet`
  fallback). `@google/generative-ai ^0.24.1` remains a dependency for it.
- **Trap for porters:** `gemini-retry.js` is a misleadingly-*named* generic
  retry helper — 12 files import it, but the main AI layer is **Claude**, not
  Gemini. Do not infer a Gemini main layer from the import graph.
- Not re-verified: the `§3.2` "no `cache_control` / `thinking` / streaming"
  absence claims.

### §3.3 Data layer — STALE → corrected

- `BUILD-PLAN` claimed the catalogue was `data/cuisines.js`. **No such file
  exists.** The catalogue is **`cuisines-vault.js` at the gia repo root**; its
  source of truth is **`doc/Feature/cuisines_js.MD`**.
- Each `vault/<version>/` is a **full repo snapshot** (CLAUDE.md, the engine
  `.js`, `data/`, `doc/`…), not just venue data — which is why 16 retained
  versions total 766 MB.
- **Applied:** `§3.3` rewritten; the `data/cuisines.js` references in `§8.1`
  Options A/B/C corrected to `cuisines-vault.js`; the `§8.1` "wired" note's
  sparse-checkout target refined.

### §3.4 — stale bullet corrected

`§3.4` still listed the Gemini R.E.D path as "not ported." ADR-002 (merged
2026-05-22) brought it into scope as Phase 5. The bullet was updated to point
at ADR-002.

### §3a engine inventory — ACCURATE

84 root `.js` files, against `~85` claimed.

## Side-finding — bridge hygiene

The submodule pulls `vendor/soleat/CLAUDE.md` and `vendor/soleat/doc/CLAUDE.md`
into the working tree. A future Gia-WA session could mistake gia's doc contract
(serial numbers, the AU Recipe, time-anchor protocol) for nested instructions.
**Recommendation:** add a line to Gia-WA's root `CLAUDE.md` stating that
`vendor/` is the read-only `ang-kl/gia` submodule and its `CLAUDE.md` files are
*not* Gia-WA's contract. (Not done in this pass — flagged for operator.)

was required — ADR-003 already covers the bridge itself.
